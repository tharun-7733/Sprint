## Sprint Goal

Deliver a fully populated and validated SQLite database (`nifty100.db`) containing data from
all 12 source Excel files across 11 database tables, with all 16 Data Quality rules implemented
and zero CRITICAL failures.

---

## Quick Start

```bash
# 1. Clone & enter project
cd Sprint/

# 2. Create virtual environment & install dependencies
make setup

# 3. Generate synthetic Excel data (or place real files in data/raw/)
python scripts/generate_synthetic_data.py

# 4. Run the full ETL pipeline → nifty100.db
make load

# 5. Run all tests
make test

# 6. Generate DQ report
make report

# 7. Launch dashboard
make dashboard

# 8. Launch REST API
make api
```

## Data Quality Rules

| Rule   | Description                                  | Severity |
|--------|----------------------------------------------|----------|
| DQ-01  | Primary key uniqueness                       | CRITICAL |
| DQ-02  | (company_id, year) composite uniqueness      | CRITICAL |
| DQ-03  | Foreign key integrity                        | CRITICAL |
| DQ-04  | Balance sheet balances within 1%             | WARNING  |
| DQ-05  | Operating margin cross-check                 | WARNING  |
| DQ-06  | Positive sales (revenue > 0)                 | WARNING  |
| DQ-07  | Net cash consistency (opening→closing)       | WARNING  |
| DQ-08  | Tax rate 0%–50%                              | WARNING  |
| DQ-09  | Dividend ≤ net profit                        | WARNING  |
| DQ-10  | URL format validation                        | WARNING  |
| DQ-11  | EPS sign matches net profit sign             | WARNING  |
| DQ-12  | Assets = current + fixed (±5%)              | WARNING  |
| DQ-13  | Interest coverage ≥ 1.0x                    | WARNING  |
| DQ-14  | Duplicate row detection                      | CRITICAL |
| DQ-15  | Missing mandatory values                     | CRITICAL |
| DQ-16  | YoY revenue change ≤ 200%                   | WARNING  |

## Configuration

Copy `.env.example` to `.env` and adjust:

```dotenv
LOG_LEVEL=INFO
DATABASE_URL=nifty100.db
DATA_DIR=data/raw
OUTPUT_DIR=output
TRUNCATE_BEFORE_LOAD=true
```

---

## Using Real Data

Replace the synthetic Excel files in `data/raw/` with your actual files using the same
column names as specified in `scripts/generate_synthetic_data.py`, then run:

```bash
make clean
make load
```

The pipeline is **idempotent** and can be safely re-run at any time.
